# Lista-Compra

# Definición
Este es un proyecto destinado a crear una aplicación que permita crear y gestionar una lista de la compra. 

# Funcionamiento y ejecución
El programa de la aplicación constará de un menú de opciones que ofrecerá diferentes funcionalidades para gestionar una lista de la compra. Éstas comprenden añadir un producto a la lista, eliminar un producto, buscar un producto en la lista, mostrar la lista, vaciar lista y por último salir del programa. Al iniciar el programa, se pedirá al usuario que introduzca un número para seleccionar una de las opciones mostradas. Esto permitirá ejecutar la funcionalidad deseada, como añadir un producto, por ejemplo. Una vez que se haya ejecutado una funcionalidad, el programa pedirá introducir de nuevo un número correspondiente a una opción. Si se desea ejecutar otra función, se introducirá el número correspondiente a la misma, mientras que si ya se ha terminado y se desea salir, se introducirá el número de opción que corresponda (en este caso el 6). Una vez que se haya salido, si se quiere acceder de nuevo al programa, habrá que iniciarlo otra vez, momento a partir del cual se repetirá el mismo proceso. 

# Pruebas unitarias
El proyecto ha sido planteado para que sus métodos puedan ser testeados mediante las pruebas unitarias con JUnit. Para ello, habiendo añadido correctamente la librería correspondiente a JUnit5 (mediante la opción "Build Path" y añadiendo la extensión al "modulepath"), revisando las importaciones  y habiendo definido el test correspondiente a cada método (que este caso en su mayoría serán del tipo AssertTrue o AssertEquals),  será posible ejecutar las pruebas haciendo clic derecho en el proyecto (o clase que se quiera testear). Esto ofrecerá la opción de ejecutarlo como pruebas unitarias, deviolviendo los métodos testeados y los errores. Si además se desea realizar la cobertura de cada línea de código, se seleccionará la opción de ejecutar como Coverage, ofreciendo en verde las líneas que pasan el test, en amarillo las secciones que no han podido ser totalmente cubiertas, y en rojo las que no lo han superado.

# Commits relevantes
A continuación se detallan los commits más relevantes de este proyecto. En primer lugar, la confección de las pruebas unitarias dio problemas habituales con las importaciones, de manera que se tuvo que resolver este asunto revisando las importaciones y colocando la librería de JUnit en el "modulepath". Estos cambios se detallaron en los commits "Pruebas unitarias" y "Revisión de pruebas unitarias". Otro problema que se solventó fue la imposibilidad inicial de crear un javadoc con la documentación, resuelto al mover el "module-info.java" al paquete main, detallado en el commit "Generar javadoc". Por último, un compañero introdujo cambios en el código desde GitHub que crearon un conflicto, de modo que se resolvió haciendo "pull" del repositorio remoto al local, ajustando manualmente las líneas de código conflictivas y una vez resueltas reaalizando otro commit y haciendo "push" de nuevo al remoto. Se puede observar en el commit "Resolver conflicto".
